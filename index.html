    Introduction

    به راهنمای وب‌سرویس REST سامانه‌ی صبا نوین خوش آمدید.

    در این راهنما در مورد چگونگی کار با وب‌سرویس REST سامانه صبا نوین صحبت خواهیم کرد.

    در ابتدا باید بگوییم که در وب‌سرویس REST تمامی درخواست‌ها در قالب یک HTTP(S) Request انجام می‌شود.

    فرمت کلی هر درخواست به شکل زیر است:

    http://api.sabanovin.com/v1/YOUR-API-KEY/Region/MethodName.OutputFormat

    خروجی هر دستور بسته به نیاز شما می‌تواند با فرمت JSON یا XML باشد که باید نام فرمت دلخواه را در انتهای هر درخواست قرار دهید.

    Authentication

    مثال:

    "http://api.sabanovin.com/v1/YOUR-API-KEY/credit.json"

    "http://api.sabanovin.com/v1/YOUR-API-KEY/credit.xml"

    در مثال فوق بجای YOUR-API-KEY باید API-Key واقعی خود را قرار دهید.

    وب‌سرویس REST سامانه صبا نوین از API-Key برای وصل شدن به سیستم API خود استفاده می‌کند.

    ابتدا شما باید یک API-Key از بخش بخش وب‌سرویس سامانه ایجاد کنید و سپس در تمامی درخواست‌ها باید API-Key خود را در آدرس URL وارد نمایید.

    http://api.sabanovin.com/v1/YOUR-API-KEY/method-name

    بجای YOUR-API-KEY باید API-Key واقعی خود را قرار دهید.

    پارامتر‌های ورودی

    نام نوع توضیحات
    gateway الزامی شماره درگاه ارسالی(شماره خط)
    to الزامی شماره گیرندگان
    text الزامی متن پیام
    برای تعداد بیش از یک گیرنده، شماره ها را با کاما , از هم جدا کنید.

    پارامتر‌های خروجی

    number شماره دریافت کننده
    status وضعیت
    datetime تاریخ ارسال (میلادی)
    datetime_fa تاریخ ارسال (شمسی)

    از این متد برای دریافت پیامک‌های دریافتی استفاده کنید.

    پارامتر‌های ورودی

    نام نوع توضیحات
    gateway الزامی شماره درگاه(شماره خط)
    from_date اختیاری دریافت شده از تاریخ
    to_date اختیاری دریافت شده تا تاریخ
    is_read اختیاری پیام‌های خوانده شده یا نشده(پیش‌فرض خوانده نشده)
    پیام‌های خوانده‌شده پیام‌هایی هستند که قبلا یا از طریق وب‌سرویس و یا از طریق وب‌هوک آنها را دریافت کرده‌اید.

    پارامتر‌های خروجی

    نام توضیحات
    reference_id شناسه پیامک
    number شماره ارسال کننده
    datetime تاریخ ارسال (میلادی)
    datetime_fa تاریخ ارسال (شمسی)

    StatusTable

    جدول وضعیت پیام‌ها:

    وضعیت توضیح
    ENQUEUE پیام در صف قرار گرفته است
    DELIVERED پیام تحویل داده شده
    FAILED خطا در ارسال پیام - برگشت هزینه
    BLACKLIST شماره گیرنده در بلک‌لیست است - برگشت هزینه
    UNDELIVERED پیام هنوز به گیرنده تحویل داده نشده(در دسترس نبودن)
    SCHEDULED پیام زمانبندی شده
    INVALID_NUMBER شماره گیرنده اشتباه - برگشت هزینه
